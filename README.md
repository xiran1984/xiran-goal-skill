# xiran-goal

> 把任何目标拆成条件树，找到瓶颈，走最短可验证路径。
> 不问"怎么做更好看"，只问"哪里卡住了，下一步做什么"。

## 什么时候用这个技能

当你遇到以下任一情况时，直接用 `/xiran-goal` 开头说话：

- **目标明确但路径模糊** — 比如"我想做自媒体但不知道从哪开始"，你知道要去哪，但不知道先迈哪只脚
- **想法很多但无法收敛** — 脑子里有七八个方向，每个都想做，结果一个都没推进
- **反复纠结但迟迟不动** — 总觉得信息不够、准备不足，想了三个月还没第一步
- **遇到瓶颈不知道卡在哪** — 事情推不动，但说不清楚是能力问题、资源问题、还是环境问题
- **需要做决策但变量太多** — 比如选 offer、选方向、选合作伙伴，每个选项都有利弊，比较不出来
- **想验证一个想法是否靠谱** — 不是要完美的分析报告，而是要一个可执行的下一步

**一句话：当你的脑子是一个毛线球，你需要有人帮你找到线头。**

## 怎么用

在 Claude Code 中，用 `/xiran-goal` 开头描述你的目标：

```
/xiran-goal 我想在3个月内从零开始做一个能赚钱的AI工具产品
```

你会得到五样东西：

| 产出 | 是什么 | 例子 |
|------|--------|------|
| **Final Goal** | 一句话说清楚目标 | "在3个月内上线一个能产生首次付费的AI工具" |
| **Assumptions** | 当前已知的约束条件 | 假设你每天有2小时业余时间 |
| **Condition Tree** | 把目标拆成一层层的必要条件 | 需求验证 → 技术选型 → MVP构建 → 冷启动 → 收费 |
| **Priority Path** | 按依赖关系排好的行动顺序 | 先验证需求再写代码，而不是一上来就买域名 |
| **Verification** | 怎么判断这条路走对了 | 第一周有3个人愿意为你的方案付费 |

### 每次问自己三个问题

拿到输出后，问自己：

1. **当前瓶颈是哪个分支？** — 别的都可以先不管，就盯着这一个
2. **哪个分支下一步动作最明显？** — 能一分钟动手的不要花十分钟规划
3. **哪个分支失败了代价最大？** — 先给它加一道保险

## 可以问什么

以下九个方向是观察身边牛人总结出的关键能力，直接问即可：

```
/xiran-goal 我如何发现并变现信息差？
/xiran-goal 我如何做出能给人背书的作品？
/xiran-goal 我如何把个人能力固化为标准化流程？
/xiran-goal 我如何做到无条件自信（厚脸皮）？
/xiran-goal 我如何用自媒体输出内容建立影响力？
/xiran-goal 我如何有效链接人脉？
/xiran-goal 我如何建立上台演讲的能力？
/xiran-goal 我如何塑造和包装个人形象？
/xiran-goal 我如何持续沉淀个人资产？
```

每个问题都会得到：目标定义 → 条件树拆解 → 优先路径 → 验证标准。

形象包装方面，一个经典案例是黄仁勋——他二十年来几乎只穿黑色皮夹克。这不是个人审美，而是一个精确计算的个人品牌策略：(1) 消除每日着装决策，节省认知资源；(2) 在一群穿西装的CEO中制造独一无二的视觉锚点；(3) 皮夹克传达的信号是叛逆者、创新者，与NVIDIA品牌定位一致；(4) 极简形象在所有场合复用，管理成本趋近于零；(5) 持续二十年不变，时间越长品牌复利越大。核心启示：形象不是更好看，而是更容易被识别、记住、信任。选一个锚点，然后坚持到它变成你的专属符号。

## 核心原理

底层是同一个执行框架：

```text
任何目标
├─ 第一步：解构 → 这个目标的必要条件是什么？（条件树）
├─ 第二步：定位 → 当前卡在哪个条件上？（瓶颈识别）
├─ 第三步：执行 → 解决这个瓶颈的最短路径是什么？（优先路径）
├─ 第四步：验证 → 我怎么确定问题真的解决了？（验证标准）
└─ 第五步：迭代 → 下一个瓶颈是什么？回到第一步
```

这不是思维模型，是执行框架。价值不在于看懂，在于用起来。
