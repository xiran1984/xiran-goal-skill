# xiran-goal

> 把任何目标拆成条件树，找到瓶颈，走最短可验证路径。
> 不问"怎么做更好看"，只问"哪里卡住了，下一步做什么"。

## 什么时候用这个技能

当你遇到以下任一情况时，直接用 `/xiran-goal` 开头说话：

- **目标明确但路径模糊** — 比如"我想做自媒体但不知道从哪开始"，你知道要去哪，但不知道先迈哪只脚
- **想法很多但无法收敛** — 脑子里有七八个方向，每个都想做，结果一个都没推进
- **反复纠结但迟迟不动** — 总觉得信息不够、准备不足，想了三个月还没第一步
- **遇到瓶颈不知道卡在哪** — 事情推不动，但说不清楚是能力问题、资源问题、还是环境问题
- **需要做决策但变量太多** — 比如选 offer、选方向、选合作伙伴，每个选项都有利弊，比较不出来
- **想验证一个想法是否靠谱** — 不是要完美的分析报告，而是要一个可执行的下一步

**一句话：当你的脑子是一个毛线球，你需要有人帮你找到线头。**

## 怎么用

在 Claude Code 中，用 `/xiran-goal` 开头描述你的目标：

```
/xiran-goal 我想在3个月内从零开始做一个能赚钱的AI工具产品
```

你会得到五样东西：

| 产出 | 是什么 | 例子 |
|------|--------|------|
| **Final Goal** | 一句话说清楚目标 | "在3个月内上线一个能产生首次付费的AI工具" |
| **Assumptions** | 当前已知的约束条件 | 假设你每天有2小时业余时间 |
| **Condition Tree** | 把目标拆成一层层的必要条件 | 需求验证 → 技术选型 → MVP构建 → 冷启动 → 收费 |
| **Priority Path** | 按依赖关系排好的行动顺序 | 先验证需求再写代码，而不是一上来就买域名 |
| **Verification** | 怎么判断这条路走对了 | 第一周有3个人愿意为你的方案付费 |

### 每次问自己三个问题

拿到输出后，问自己：

1. **当前瓶颈是哪个分支？** — 别的都可以先不管，就盯着这一个
2. **哪个分支下一步动作最明显？** — 能一分钟动手的不要花十分钟规划
3. **哪个分支失败了代价最大？** — 先给它加一道保险

## 真实场景示例

以下示例来自我观察到的身边牛人的实际做法。

---

### 1. 发现信息差

> **信息差的本质不是"知道别人不知道的"，而是"在别人还没重视的地方，你提前布局了认知和资源"。**

**条件树片段：**

```text
发现并变现信息差
├─ 输入端：持续把自己暴露在信息浓度高的地方
│  ├─ 关注一手源而非二手解读（论文、官方文档、海外原帖）
│  ├─ 关注交叉领域（两个行业的交界地带，比如AI+法律、AI+教育）
│  ├─ 关注"现在不起眼但增速异常"的信号（小众工具突然爆发式增长）
│  └─ 建立关键词监控，每周扫一次
├─ 加工端：把信息变成可交易的认知
│  ├─ 用自己做过的事来验证信息，而非只转发
│  ├─ 把零散信息结构化（整理成清单、流程图、决策树）
│  └─ 写出"如果这个趋势继续，什么人会受益？什么生意会消失？"
└─ 变现端：让信息差兑现
   ├─ 把加工后的认知打包成可交付的产品（报告、课程、工具、咨询）
   ├─ 找到最需要这个信息但自己拿不到的人
   └─ 在信息差被填平之前完成第一波变现
```

**牛人怎么做的：** 早期跨境电商卖家不是英语最好的人，而是最先发现"1688进货+亚马逊加价=巨大差价"这个简单事实的人。他们不是天才，只是比别人早半年看到了这个等式，然后在别人还在观望时把货发到了美国仓库。

### 2. 做出背书作品

> **背书作品不是你觉得好的作品，而是别人愿意用它来向第三个人介绍你的作品。**

**条件树片段：**

```text
做出真正的背书作品
├─ 选对战场：找一个别人可以通过一个作品就判断你水平的领域
│  ├─ 避免需要长篇解释的领域（"你要先理解我的思路才能看懂"= 无效背书）
│  └─ 选那种结果可直观对比的领域（设计稿、代码仓库、数据报告、公开演讲）
├─ 做对产品：作品本身必须过一道"外人评判关"
│  ├─ 让不认识的同行看了会点头，而不只是朋友说好
│  ├─ 包含至少一个让人想截屏传播的片段
│  └─ 附带"为什么这么做"的方法论，而不只是结果展示
├─ 放对地方：作品要出现在决策者会搜索的位置
│  ├─ GitHub、个人网站、行业社区、搜索引擎第一页
│  └─ 当别人搜你的领域关键词时，你的作品排在前面
└─ 有人引用：最终验证不是你的阅读量，而是
   ├─ 有人在他们的文章里引用了你的作品
   ├─ 有人在面试时提到了你的作品
   └─ 有人因为你的作品主动来找你合作
```

**牛人怎么做的：** 一个程序员花两周写了一个开源工具解决自己的痛点，写好 README 和 demo 视频发布到 GitHub，在 Hacker News 上被顶到首页。三个月后，他收到了三家公司的 offer，因为对方 CTO 都是这个工具的用户。他的代码不是最漂亮的，但他的作品出现在了对的人面前。

### 3. 固化标准化

> **牛人和普通人的核心区别之一：牛人遇到重复问题会写SOP，普通人每次重新动脑。**

**条件树片段：**

```text
把个人能力固化为可复用的系统
├─ 识别标准化对象
│  ├─ 你一周做超过两次的事 → 值得写流程
│  ├─ 你教过别人超过一次的事 → 值得写文档
│  ├─ 你犯过超过一次的错误 → 值得写检查清单
│  └─ 你做决策时反复权衡的因素 → 值得写决策树
├─ 写出第一版（不求完美，求有）
│  ├─ 用最简单的工具：Notion、飞书、甚至一个txt文件
│  ├─ 格式：第一步做什么→第二步做什么→常见坑→完成标准
│  └─ 写完后马上用自己的流程重做一遍，边做边改
├─ 让别人能用
│  ├─ 找一个完全不了解这件事的人，让他照着做一次
│  ├─ 他卡在哪里，哪里就需要补充
│  └─ 持续迭代直到新人照着做能独立完成
└─ 把SOP变成你的资产
   ├─ 这个SOP本身就是你能力的外部证明
   ├─ 当你可以把这件事交给别人时，你就能腾出手做更高价值的事
   └─ 多个SOP叠加=一个可复制的小系统=你可以不做执行者
```

**牛人怎么做的：** 一个运营每次做活动都自己忙到凌晨。后来他把活动拆成"活动前7天检查清单+活动当天时间轴+活动后复盘模板"，招了一个实习生照着做。第一次实习生做错了30%，他改了清单；第二次只错10%；第三次之后他再也不用自己做了。他把省下的时间用来学投放，半年后从运营变成了投放负责人。

### 4. 无条件自信（厚脸皮）

> **无条件自信不是"我很牛逼"的幻觉，而是"无论结果如何，我对自己说的话能做到"的自我兑现记录。**

这是你刚才问的问题，我把完整的条件树放在这里：

```text
无条件自信
├─ 切除条件依赖：先看清自信目前绑在哪些易碎的东西上
│  ├─ 列举：我的自信什么时候会塌？（被否定时？失败时？对比不如人时？不被喜欢时？）
│  ├─ 对每条写下：这个条件消失时，我这个人还值不值得存在？（答案永远是"值得"）
│  └─ 这一步不是说服自己，是看清楚依赖结构，后续每步会逐个拆掉
│
├─ 建立自证闭环：让自信的来源从"外部验证"切换到"自我兑现"
│  ├─ 核心机制：自信 = 我对自己说要做某事的相信程度 × 我实际去做了的兑现率
│  │  ├─ 每天对自己做一个极小的承诺（如"今天写200字"），然后做到
│  │  ├─ 做不到时不自我攻击，而是降低承诺难度直到能持续兑现
│  │  └─ 持续 2-4 周，自我兑现率上升，自信的根基从"别人怎么看我"转移到"我能否信自己"
│  ├─ 区分"我能控制的"和"我不能控制的"
│  │  ├─ 我能控制：我的投入质量、我的判断过程、我是否守约、我是否复盘
│  │  ├─ 我不能控制：结果、他人评价、运气、市场反馈、竞争结果
│  │  └─ 把自信锚定在可控域，不可控域只作为信息输入而非自我评判依据
│  └─ 建立独立于角色的自我定义
│     ├─ 当"我是谁"绑定在某个身份（职位、收入、关系状态）上时，失去它自信就崩塌
│     └─ 定义自己为"一个在持续行动和迭代的人"，而非"一个拥有X头衔/X资产的人"
│
├─ 提高失败容错率：让一次失败不再威胁自我评价
│  ├─ 增加尝试频率以降低单次 stakes
│  │  ├─ 一年只做一件事，失败就是全年失败 → 压力巨大
│  │  └─ 一个月做多件事，失败一件其他还在跑 → 单次失败不定义你
│  ├─ 建立"失败 → 提取信息 → 调整 → 再试"的肌肉记忆
│  │  └─ 经历过足够多次"失败但没有毁灭"的循环，大脑自然脱敏
│  └─ 财务安全垫：6个月生活费的现金缓冲
│     └─ 很多"不自信"其实是经济脆弱性的心理投射，不是性格问题
│
├─ 切断社会比较回路：不靠比别人好来确认自己
│  ├─ 限制被动比较输入：减少无目的刷社交媒体
│  ├─ 把比较方向从"他人 vs 我"改为"昨天的我 vs 今天的我"
│  └─ 在强者中不自卑，问"我能学到什么"而非"我为什么不如他们"
│
├─ 身体底层支撑：生理状态差会系统性地压低自信基线
│  ├─ 睡眠不足 → 情绪调节能力下降 → 任何挫折都会被放大
│  ├─ 缺乏运动 → 身体感受差 → 自我意象降低
│  └─ 不是要完美，但要保证底线不崩
│
└─ 防回退机制
   ├─ 警惕条件：长期孤立、持续重大失败、toxic环境、成瘾行为
   ├─ 定期自检：最近一周，我的自信有没有重新绑上某个条件？
   └─ 发现回退时，回到第一步重新拆解，不要自责
```

**牛人怎么做的：** 一个销售新人前三个月一单没签，同事都在出单。他没有自我否定，而是把每一次被拒绝后的自我对话从"我不适合做销售"改成"这个客户教会了我什么"。他把每次拒绝的原因记下来，整理了50个常见拒绝理由和应对话术。第四个月他成了团队的销冠。他的自信不是来自"我一定能签单"，而是来自"我被拒绝后还能继续优化"。

### 5. 自媒体输出内容

> **自媒体不是"运营一个账号"，而是"把你的认知变成可传播的资产"。内容本身会成为你的24小时销售。**

**条件树片段：**

```text
用自媒体输出建立影响力
├─ 定位：找一个你讲得清楚、别人搜得到的交叉点
│  ├─ 不能只选你擅长的，要选别人愿意搜的
│  ├─ 公式：你的专业领域 + 大众关心的话题 = 你的定位
│  │    例如：财务 + AI工具 = "用AI帮你省税的会计师"
│  │    例如：前端 + 独立开发 = "教程序员一个人做出能赚钱的产品"
│  └─ 检验标准：你能不能在10秒内向一个陌生人讲清楚你写什么、写给谁看
├─ 生产：建立持续输出的流水线，不靠灵感
│  ├─ 维护一个"选题库"，随时记录，永远有下一条可写
│  ├─ 把一条内容拆成多个形式：长文→短文→图文→视频脚本
│  ├─ 固定节奏，宁可频率低也不要忽冷忽热（每周1篇稳定 > 每天1篇连发两周然后消失）
│  └─ 每篇内容回答一个问题，而不是表达一个观点
├─ 分发：让内容出现在目标人群的眼睛前
│  ├─ 选择1-2个主平台深挖，不要同时铺5个平台每个都浅
│  ├─ 研究平台算法偏好（什么标题被点击、什么内容被收藏）
│  ├─ 在同类大号的评论区提供有价值的补充，吸引精准关注
│  └─ 主动把自己的内容发给3个可能受益的人，请他们转发
└─ 转化：让关注变成机会
   ├─ 在内容中留下"下一步"的钩子（加微信、领资料、看demo）
   ├─ 定期整理读者的高频问题，这些问题就是你的产品方向
   └─ 当有人因为你半年前的一篇文章来找你时，复利开始了
```

**牛人怎么做的：** 一个产品经理每周写一篇"我用AI工具解决了什么问题"的实战记录，不讲理论只讲操作。前三个月阅读量很低，但他坚持写。第六个月有一篇被大号转载，涨了5000粉。一年后有人因为看到他的文章邀请他去做AI产品咨询，单次咨询费超过他月薪。他用52篇文章换了一个副业。

### 6. 链接人脉

> **有效人脉不是"认识很多人"，而是"很多人知道你具体能做什么，并且在需要时想到你"。**

**条件树片段：**

```text
建立有效人脉网络
├─ 降低被发现的成本：让别人能搜到你、看懂你
│  ├─ 有一个能说明你做什么的个人主页/账号/作品集
│  ├─ 当别人搜你的名字时，前三条结果能说明你的专业领域
│  └─ 你的自我介绍不是"XX公司XX职位"，而是"我帮XX人解决XX问题"
├─ 创造被需要的理由：给别人一个联系你的抓手
│  ├─ 持续输出专业内容（见上一条"自媒体输出"）
│  ├─ 主动帮助你能帮的人，不求立刻回报
│  │    例如：看到一个创业者在找技术合伙人，你虽然不合适但帮他转发了
│  └─ 每次与人见面结束后，24小时内发一条有用的资料给对方
├─ 进入高价值场域：人和人的差距是被环境拉开的
│  ├─ 找到你所在领域前20%的人在哪里聚集（社群、会议、圈子）
│  ├─ 先贡献价值再建立关系（分享见解、帮忙整理资料、做志愿者）
│  ├─ 每次活动后只加3个人，但要和每个人约一次15分钟的后续交流
│  └─ 一年下来你会有30个真正认识你、信任你的高质量连接
└─ 维护：让关系不冷却
   ├─ 每季度给重要联系人发一次更新（你最近在做什么，你学到了什么）
   ├─ 看到对某个人有用的信息随手转发给他
   └─ 只维护那些双向愿意投入的关系，单向索取型关系不维护
```

**牛人怎么做的：** 一个技术博主每次参加行业会议后，当晚写一篇参会笔记发给演讲嘉宾，附上"你讲的那个XX观点我特别有共鸣，这是我在这方面的补充思考"。很多人回复了他，有几个成了长期合作者。他没有求任何人帮忙，但他成了"那个每次会后都会认真写笔记的人"，机会开始主动找他。

### 7. 上台演讲

> **公开演讲是个人影响力的加速器。一次好的演讲可以让100个人在30分钟内信任你，这比你一对一聊三个月都有效。**

**条件树片段：**

```text
建立公开演讲能力
├─ 准备阶段：好的演讲90%的工作在登台前完成
│  ├─ 明确一句话核心信息：听众离开后只能记住一件事，那件事是什么？
│  ├─ 为你的听众定制内容：他们在乎什么？他们的痛点是什么？
│  ├─ 用故事承载观点：一个亲身经历的故事 > 十个抽象的道理
│  │    结构：冲突/困境 → 尝试 → 转折 → 收获
│  ├─ 准备比所需时间多30%的内容，但准备提前结束的方案
│  └─ 至少完整演练3次，最后一次要录音/录像
├─ 临场：控制你能控制的，接受你不能控制的
│  ├─ 前30秒定生死：用一个问题、一个事实或一个故事开场，不要自我介绍
│  ├─ 把演讲当成"和台下的朋友聊天"而非"向观众表演"
│  ├─ 语速放慢到你感觉有点慢的程度 —— 对听众来说刚刚好
│  ├─ 沉默3秒比"嗯啊然后"好一万倍
│  └─ 有人看手机不代表你讲的不好，这是人性
├─ 进阶：从"讲完"到"讲出影响"
│  ├─ 每次演讲后找一个人问"你记住了什么"，看是不是你设定的一句话核心
│  ├─ 积累5个你可以随时讲好的故事（职业转折、失败教训、关键选择）
│  ├─ 主动寻找分享机会：公司内部分享、行业meetup、线上直播
│  └─ 录制你的演讲并发布（回到"自媒体输出"那条）
└─ 机会获取：你不争取就没有人给你舞台
   ├─ 从最小的场合开始：团队分享 → 部门分享 → 公司分享 → 外部活动
   ├─ 主动报名：大多数活动的演讲名额没有你想象的竞争激烈
   └─ 每次讲完后把演讲稿/PPT优化为线上可传播的版本，增加被二次邀请的概率
```

**牛人怎么做的：** 一个不太会说话的程序员强迫自己每季度在公司内部做一次技术分享。第一次他手抖到拿不稳遥控器，照着PPT念完了。第三次他开始能看台下的人了。第五次他被一个参会的外部嘉宾邀请去行业大会演讲。他说秘诀就一条："把你要讲的内容缩减到三个点，每个点配一个你亲身经历的故事，然后提前对着镜子讲五遍。"

### 8. 塑造包装形象

> **形象不是肤浅的外表，而是降低别人理解你的成本。在你开口之前，别人已经通过你的形象对你有了判断。**

**条件树片段：**

```text
建立有辨识度的个人形象
├─ 视觉锚点：让人看一眼就能记住的一个特征
│  ├─ 可以是一件标志性单品（黑T恤、特定颜色的眼镜）
│  ├─ 可以是一种固定风格（极简、复古、运动）
│  ├─ 关键是"始终如一"——变来变去等于没有形象
│  └─ 选择标准：这个形象是否准确传达了你想要的职业信号？
├─ 言语锚点：你反复说的话会变成你的标签
│  ├─ 找到一句能代表你价值观的话，在合适的场合反复说
│  │    例如："先完成再完美""数据不说谎""用户不会骗人"
│  ├─ 不是喊口号，而是要每次配上你真实做过的事来佐证
│  └─ 当别人转述你的这句话时，你的个人品牌就成立了
├─ 行为锚点：你怎么做事，就是你的品牌
│  ├─ 你是不是一个"说到做到"的人 → "靠谱"标签
│  ├─ 你是不是一个"能化简为繁"的人 → "清晰"标签
│  ├─ 你是不是一个"每次都能多想一步"的人 → "用心"标签
│  └─ 你的行为标签最终会超越你的职位标签
└─ 一致性：形象崩塌的原因永远不是"不够好"，而是"不一致"
   ├─ 你说自己重视质量，但每次交付都有低级错误 → 形象崩塌
   ├─ 你说自己讲真话，但每次遇到冲突就回避 → 形象崩塌
   └─ 形象 = 别人对你的预期。满足预期 = 靠谱。超出预期 = 难忘
```

#### 黄仁勋为什么一直穿黑色皮夹克？

黄仁勋（Jensen Huang，NVIDIA CEO）在几乎所有公开场合都穿着同一件黑色皮夹克。这不是偶然，而是一个精心设计的个人形象策略：

1. **降低决策成本** — 像乔布斯的高领毛衣、扎克伯格的灰T恤一样，固定着装消除了"今天穿什么"这个日常决策。顶级决策者深知决策疲劳是真实存在的，把精力的配额留给真正重要的事。

2. **制造视觉锚点** — 在一群穿西装的CEO中，一个穿皮夹克的人立刻被记住。黑色皮夹克成为黄仁勋的视觉商标，看到皮夹克就想到他，看到他就在找皮夹克。这种双向绑定是无价的个人IP。

3. **传达品牌信号** — 黑色皮夹克传达的信号是：我不是传统商人，我是叛逆者、创新者、技术狂热者。这和NVIDIA的品牌定位高度一致——不是跟随者，是定义规则的人。他的个人形象 = 公司形象的具象化。

4. **可复制性** — 一个极简的形象可以在任何场合复用：发布会、采访、内部会议、行业论坛。不需要造型师、不需要根据场合调整，形象管理成本趋近于零。

5. **持续积累** — 如果他穿了10年西装突然换成皮夹克，那是哗众取宠。但穿了20年皮夹克，这就成了一种仪式感、一种精神符号。时间越长，这个形象锚点的"品牌复利"就越大。

**核心启示：** 形象不是"变得更好看"，而是"更容易被识别、记住、信任"。选一个和你价值观一致的视觉锚点，然后坚持用到它变成你的专属符号。不要频繁换风格，一致性 > 多样性。

### 9. 沉淀资产

> **资产和打工的区别：打工是你停下来收入就停，资产是你停下来它还在运转。**

**条件树片段：**

```text
持续沉淀个人资产
├─ 认知资产：把你的知识从脑子里搬出来
│  ├─ 写文档、写SOP、写决策记录 → 形成你的方法论库
│  ├─ 每次做完一个项目，留一份"如果重新做我会怎么改"的复盘
│  ├─ 把散乱的笔记定期整理为结构化的知识库
│  └─ 你的方法论库 = 你未来咨询/教学/出书的内容储备
├─ 作品资产：每一个项目都应该留下一个可展示的成果
│  ├─ 代码 → GitHub仓库（写好README，录好demo）
│  ├─ 设计 → 作品集网站（每个案例附上问题+过程+结果，而不只是美图）
│  ├─ 文字 → 博客/公众号/小报童（文章之间互链，形成体系）
│  ├─ 数据 → 可复用的模板和模型
│  └─ 每完成一个项目都问：如果3个月后有陌生人看到这个，他会觉得有价值吗？
├─ 关系资产：把人脉变成可索引的网络
│  ├─ 维护一个联系人清单，标注每个人的领域、擅长的、你帮过他的、他帮过你的
│  ├─ 主动做"连接器"：把两个彼此不认识但可以合作的人介绍到一起
│  ├─ 当你自己成为信息流动的节点，你的关系资产就开始产生复利
│  └─ 不要只结交比你强的人。比你年轻的人是你未来的合作伙伴
├─ 产品资产：把你的一次性服务变成可重复销售的产品
│  ├─ 你的咨询能力 → 录制成课程 → 卖1000份不增加边际成本
│  ├─ 你的操作SOP → 打包成模板/工具 → 别人付费下载自己用
│  ├─ 你的行业洞察 → 写成付费报告/Newsletter → 订阅制收入
│  └─ 判断标准：如果你今天请假一周，这个资产还在为你赚钱吗？
├─ 财务资产：从"用时间换钱"到"用钱换时间"
│  ├─ 第一阶段：存够6个月生活费的现金缓冲
│  ├─ 第二阶段：现金流资产（定期存款利息、租金、分红等被动收入）
│  ├─ 第三阶段：用被动收入覆盖基本生活开支 → 获得真正的选择权
│  └─ 每一个阶段的目标都不是"更多钱"，而是"更多说不的权利"
└─ 身体资产：所有其他资产的基础设施
   ├─ 有规律的睡眠、饮食、运动是所有资产的前提
   ├─ 一个35岁身体崩了的人，前面积攒的所有资产都会贬值
   └─ 每周投入在身体上的时间，是对未来资产回报率的投资
```

**牛人怎么做的：** 一个乙方设计师每完成一个项目，除了交付给客户的文件，还会私下做三件事：(1) 把项目中的可复用组件抽象成自己的模板库，(2) 写一篇"这个项目我学到了什么"的复盘，(3) 精选最好的几个方案更新到作品集。三年后，当其他设计师还在每个项目从零开始时，他的模板库让他一周能干别人三周的工作量。同时他的作品集让他在几乎没有主动找客户的情况下持续接到私单。他把自己的时间和手艺，变成了可复用的资产。

## 核心原理

所有九个例子的底层逻辑是同一个条件树框架：

```text
任何目标
├─ 第一步：解构 → 这个目标的必要条件是什么？（条件树）
├─ 第二步：定位 → 当前卡在哪个条件上？（瓶颈识别）
├─ 第三步：执行 → 解决这个瓶颈的最短路径是什么？（优先路径）
├─ 第四步：验证 → 我怎么确定问题真的解决了？（验证标准）
└─ 第五步：迭代 → 下一个瓶颈是什么？回到第一步
```

这不是一个"思维模型"——这是一个执行框架。它的价值不在于看懂，在于用起来。
